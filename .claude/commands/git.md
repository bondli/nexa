# Git 智能提交

分析代码更改并自动生成符合规范的提交信息，然后执行 git 提交。

## 执行步骤

### 1. 检查工作区状态
首先运行 `git status` 查看当前更改状态。

### 2. 查看更改内容
- 运行 `git diff` 查看未暂存的更改
- 运行 `git diff --cached` 查看已暂存的更改
- 如果没有任何更改，提示用户并结束

### 3. 暂存更改（如需要）
如果没有已暂存的更改但有未暂存的更改：
- 询问用户是否要暂存所有更改
- 用户确认后执行 `git add -A`

### 4. 生成提交信息
根据更改内容生成提交信息，格式遵循项目约定：
<类型>: <描述>

类型包括：
-`feat` - 新功能
-`fix` - Bug 修复
-`docs` - 文档更新
-`style` - 代码格式（不影响逻辑）
-`refactor` - 重构（既不是新功能也不是修复）
-`test` - 测试相关
-`chore` - 构建/工具/依赖更新
-`perf` - 性能优化
-`ci` - CI/CD 配置

提交信息要求：
- 使用中文描述
- 简洁明了，不超过 50 字符
- 使用动词开头（如"添加"、"修复"、"更新"）

### 5. 执行提交
使用生成的提交信息执行 `git commit -m "提交信息"`

### 6.推送远程仓库
推送远程仓库，执行：`git push origin <当前分支>` 推送更改

### 7. 显示提交结果
运行 `git log --oneline -1` 显示最新提交

## 注意事项

- 如果遇到合并冲突，提示用户手动解决
- 如果 pre-commit hook 失败，显示错误信息
- 确保 `.env` 等敏感文件不会被提交
