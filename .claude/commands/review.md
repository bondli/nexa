# 代码审查命令

请对当前项目进行全面的代码审查，使用多代理协作模式：

## 🎯 审查任务分配

### 代理 1：代码质量审查
- 检查命名规范（变量、函数、组件）
- 检查注释完整性
- 检查代码复杂度（函数是否过长）
- 检查是否有未使用的代码

### 代理 2：安全检查
- SQL 注入风险
- XSS 跨站脚本风险
- 敏感信息泄露（密码、API Key）
- 不安全的依赖版本

### 代理 3：测试覆盖审查
- 关键功能是否有单元测试
- 边界情况是否覆盖
- 测试用例是否充分

### 代理 4：性能审查
- N+1 查询问题
- 内存泄漏风险
- 不必要的重复计算
- 大列表渲染优化

## 📋 输出格式

请用表格格式输出审查报告，包括：
- 每个代理的审查结果
- 问题等级（严重/警告/建议）
- 文件位置
- 修复建议
- 总体评分（10 分制）
