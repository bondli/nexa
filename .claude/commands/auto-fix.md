# 自动修复循环（Ralph Wiggum Loop）

这是一个自动循环开发任务，请按照以下流程执行，直到任务完成：

## 🔄 循环流程

### 步骤 1：分析任务
- 理解用户需求
- 列出需要修改的文件
- 制定实施计划

### 步骤 2：执行修改
- 逐个文件修改
- 每次修改后保存

### 步骤 3：自动验证
运行以下命令验证：
- npm run build（构建验证）
- npm run test（测试验证）
- npm run lint（代码检查）

### 步骤 4：检查结果
- ✅ 如果全部通过 → 进入步骤 5
- ❌ 如果有错误 → 分析错误原因，返回步骤 2 修复

## 🛑 停止条件

满足以下任一条件时停止循环：
1. ✅ 所有验证通过，任务完成
2. ⚠️ 达到最大迭代次数（5 次）
3. 🔴 遇到无法自动修复的错误
4. 🤔 需要用户确认或决策

## 📋 输出格式

每次循环后输出进度：
- 循环进度 [X/5]
- 本次修改的文件列表
- 验证结果（构建/测试/检查）
- 下一步计划
