# 🎯 审查目标

请对当前代码进行全面 Code Review，确保其达到“生产可用级别”。

---

# 📥 输入上下文（你需要自动理解）

- 用户原始需求
- OpenSpec 生成的方案（plan）
- 当前实现代码（apply 阶段产物）

---

# 🔍 审查维度（必须逐项检查）

## 1. 功能正确性
- 是否完整实现需求
- 是否存在逻辑错误
- 是否遗漏边界情况

## 2. Bug 风险
- ts类型错误 / any类型
- 空指针 / undefined
- 异步错误（async/await、Promise）
- 错误处理缺失（try/catch）

## 3. 安全问题（必须严格）
- 是否存在注入风险（SQL / XSS）
- 是否有敏感信息泄露（token / 密码）
- 是否缺少输入校验

## 4. 性能问题
- 是否存在不必要的重复计算
- 是否有潜在性能瓶颈
- 是否有不合理的 IO / 请求
- React代码的性能问题，比如缺少memo，没有使用useCallback等

## 5. 代码质量
- 可读性
- 命名是否清晰
- 是否符合最佳实践
- 是否易于维护和扩展
