## Context

### 当前状态

- 应用已有托盘功能，包含"快速笔记"入口
- 知识库功能已完成，支持文档的 CRUD 操作
- AI 服务已集成，支持文本生成和优化
- Electron 客户端架构：main + preload + renderer

### 需求概述

用户点击托盘"截图快存"，弹出黑色粘贴框，用户粘贴截图后：
1. 显示上传进度
2. OCR 识别文字
3. AI 优化文案
4. 展示 Markdown 结果
5. 用户选择知识库并保存

## Goals / Non-Goals

**Goals:**
- 实现托盘"截图快存"入口
- 实现截图粘贴和上传功能
- 集成 OCR 文字识别
- 集成 AI 文案优化
- 实现知识库保存功能

**Non-Goals:**
- 不实现截图编辑功能
- 不实现跨平台截图快捷键（使用系统粘贴）
- 不支持多图批量处理

## Decisions

### D1: OCR 实现方案

**选择:** 使用 tesseract.js (本地 OCR)
- **原因:** 无需额外 API 费用，纯前端可运行，保护用户隐私
- **备选:** 百度 OCR API / Google Cloud Vision API（需要 API 密钥）

### D2: 截图粘贴窗口实现

**选择:** Electron BrowserWindow（无框窗口）
- **原因:** 需要监听全局粘贴事件，必须在 main process 处理
- **备选:** React 组件（无法监听全局粘贴事件）

### D3: 图片传输方式

**选择:** 复用现有上传接口 `/common/uploadImage`
- **原因:** 避免重复开发，复用已有图片上传能力
- **实现:** 前端通过 FormData 上传图片，服务端返回图片路径

### D4: AI 优化调用

**选择:** 复用现有 aiService
- **原因:** 减少重复代码，保持 AI 调用统一
- **改造:** 新增 `optimizeText` 方法

## Risks / Trade-offs

### R1: OCR 识别准确率

**风险:** tesseract.js 对中文和复杂排版识别效果可能不佳
**缓解:** 提供手动编辑入口，用户可修正 OCR 识别结果后再保存

### R2: 大图片性能

**风险:** Base64 传输大图片可能导致内存问题
**缓解:** 前端限制图片大小（如 10MB），服务端压缩后再 OCR

### R3: 窗口管理

**风险:** 粘贴窗口可能遮挡用户想要的内容
**缓解:** 窗口采用半透明背景，可通过 ESC 关闭

### R4: 粘贴事件冲突

**风险:** 全局粘贴事件可能与其他应用冲突
**缓解:** 仅在截图快存窗口激活时监听粘贴事件
