## Context

Nexa 是一款 AI 知识库桌面应用，目前支持文章、笔记的存储和搜索。用户在使用微信浏览文章时，希望能够将感兴趣的文章快速保存到 Nexa。目前缺少从微信直接分享文章到 Nexa 的能力，用户需要手动复制粘贴链接。

## Goals / Non-Goals

**Goals:**
- 支持从微信分享文章到 Nexa 临时文章
- 在临时文章列表中清晰展示文章标题和 URL
- 点击临时文章可在浏览器中打开原文章
- 临时文章支持软删除，可在回收站恢复

**Non-Goals:**
- 不支持其他 App 的分享（仅微信）
- 不支持在 Nexa 内直接阅读临时文章内容
- 不涉及文章内容的爬取或摘要生成

## Decisions

### 1. URL Scheme 设计
**决定**: 使用 `nexa://share/article?title=xxx&url=xxx` 作为 URL Scheme

**备选方案**:
- 使用自定义协议 `nexa://`（采用）
- 使用通用分享协议

**理由**: 简单直观，微信和其他 App 都支持通过 URL Scheme 唤起第三方应用。

### 1.1 微信分享接收流程
**决定**: 在 RN 端新增一个分享接收页面，用户点击"保存"按钮后才写入数据库

**流程**:
1. 用户在微信中选择"更多打开方式"→选择 Nexa 应用
2. Nexa 被唤起，打开分享接收页面（传入 title 和 url 参数）
3. 页面展示文章标题和 URL，提供"保存"按钮
4. 用户点击"保存"按钮，调用 `shareToTempArticle` 方法写入数据库
5. 保存成功后跳转到临时文章列表页

**理由**: 让用户确认后再保存，提供更好的交互体验，也方便后续扩展编辑功能。

### 2. 数据库字段设计
**决定**: 在 `TempArticle` 表中新增 `title` 字段，类型为 VARCHAR(500)

**备选方案**:
- 新增独立 `title` 字段（采用）
- 使用 JSON 格式存储 title 和 url

**理由**: 独立字段便于查询和索引，结构更清晰。

### 3. RN 端分享方法设计
**决定**: 在 `ArticleService.ts` 中新增 `shareToTempArticle` 方法，接收 title 和 url 参数

**备选方案**:
- 新增独立 service 方法（采用）
- 复用现有方法

**理由**: 职责单一，便于维护和测试。

### 4. Desktop 端点击行为
**决定**: 点击临时文章在浏览器中打开 URL，不进入详情页

**备选方案**:
- 在浏览器中打开（采用）
- 进入详情页

**理由**: 临时文章仅保存标题和 URL，无详细内容可展示。

### 5. Actions 组件行为
**决定**: 临时文章的 Actions 只显示删除按钮，恢复功能通过回收站实现

**备选方案**:
- 仅显示删除按钮（采用）
- 显示删除和恢复按钮

**理由**: 临时文章设计为"稍后阅读"，删除后可在回收站恢复，无需在列表页显示恢复按钮。

## Risks / Trade-offs

| 风险 | 描述 |  mitigation |
|------|------|-------------|
| URL Scheme 兼容性 | 部分安卓设备可能不支持 URL Scheme | 通过应用宝或其他渠道提供拉起能力 |
| 微信版本兼容性 | 微信分享到第三方 App 的能力依赖微信版本 | 文档说明支持微信版本要求 |
| 数据库迁移 | 现有 TempArticle 表新增字段需要迁移 | 使用 ALTER TABLE ADD COLUMN，无数据风险 |
