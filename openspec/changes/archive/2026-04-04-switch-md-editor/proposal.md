# Proposal: Switch to Markdown Editor

## 一、What (是什么)

将当前基于 React-Quill 的富文本编辑器替换为 Markdown 编辑器，实现更好的编辑体验和代码兼容性。

## 二、Why (为什么)

### 业务背景
- 当前使用 MyQuill 富文本编辑器，功能复杂但不够轻量
- Markdown 更适合技术文档和代码片段的编写
- 用户需要在编辑笔记时能够实时预览渲染效果

### 核心价值
1. **更好的代码体验**：原生支持代码块语法高亮
2. **更轻量**：Markdown 格式更简洁，存储更高效
3. **更灵活**：支持双向切换编辑/预览模式
4. **用户需求**：点击笔记进入编辑模式，右上角可切换预览模式

## 三、Scope (范围)

### 包含
- 选择并集成 React Markdown 编辑器组件
- 实现编辑/预览模式切换
- 没有存量数据，无需考虑迁移现有笔记数据（无需HTML → Markdown）
- 实现边输入边自动保存（节流+防抖）
- 移除原有的自动提取链接功能

### 不包含
- 笔记分类功能的修改
- 笔记搜索功能的修改

## 四、Success Criteria (成功标准)

1. 用户点击笔记后进入 Markdown 编辑模式
2. 右上角提供切换到预览模式的按钮
3. 边输入边自动保存，保存有节流防抖保护
4. 自动提取链接功能已移除
5. 编辑器界面美观，与现有 UI 风格一致
