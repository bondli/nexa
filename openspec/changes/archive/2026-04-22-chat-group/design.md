## Context

当前系统中的聊天会话（Chat）功能已经实现，但缺乏分组管理能力。用户的历史会话全部平铺展示，当会话数量增加时，难以快速定位和查找。

## Goals / Non-Goals

**Goals:**
- 实现聊天会话分组管理功能
- 支持分组创建、重命名、删除操作
- 支持会话移动到指定分组
- 未分组会话按时间维度展示（今天、昨天、七天前、更早）
- 分组支持显示会话数量统计

**Non-Goals:**
- 不涉及笔记/文章的分类功能
- 不支持分组的拖拽排序
- 不支持多级分组

## Decisions

1. **数据模型设计**
   - 新增 ChatCate 表，包含：id, name, counts, createdAt, updatedAt
   - Chat 表增加 cateId 字段，关联 ChatCate
   - counts 字段使用冗余设计，直接存储分组下的会话数量，避免频繁计算

2. **前端 UI 布局**
   - 分组标题行：显示"分组"文字，右侧新增分组入口
   - 分组列表：显示所有分组，支持展开/折叠
   - 未分组会话：按时间分组展示

3. **分组操作交互**
   - 新增分组：点击右侧 + 按钮，弹出输入框，回车确认
   - 编辑分组：右键或点击操作按钮，出现下拉菜单（重命名、删除）
   - 分组详情：点击分组打开 Modal，展示该分组下的会话列表

## Risks / Trade-offs

- [风险] counts 字段可能与实际数量不一致 → 每次增删会话时同步更新 counts
- [风险] 时间分组展示需要频繁计算 → 考虑前端缓存计算结果
