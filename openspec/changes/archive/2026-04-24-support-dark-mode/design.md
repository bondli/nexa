## Context

Nexa 是一个 AI 知识库桌面应用，前端使用 React + TypeScript + Ant Design 4。当前应用没有主题切换功能，所有样式都是硬编码的亮色模式。用户需要在不同环境下使用应用，暗色模式可以减少视觉疲劳。

参考 Ant Design 官方文档的主题切换方案：https://ant.design/docs/react/customize-theme

## Goals / Non-Goals

**Goals:**
- 实现三种主题模式：浅色模式、深色模式、跟随系统
- 在 User 组件菜单中增加"外观设置"入口
- 主题选择持久化存储
- 修改硬编码样式支持主题切换

**Non-Goals:**
- 不涉及服务端改动
- 不自定义主题配色方案，保持 Ant Design 默认配色

## Decisions

### 1. 使用 Ant Design ConfigProvider 实现主题切换

**决策**: 使用 Ant Design 的 ConfigProvider 组件和 theme 属性实现主题切换

**理由**:
- Ant Design 官方推荐方案，稳定性高
- 与现有技术栈无缝集成
- 支持亮色/暗色模式切换

**备选方案**:
- CSS 变量方案：需要大量修改现有组件样式，工作量大

### 2. 主题配置存储方案

**决策**: 使用 localStorage 存储主题偏好

**理由**:
- 简单直接，无需服务端配合
- 适用于桌面应用场景

**备选方案**:
- electron-store：需要引入额外依赖

### 3. 主题切换入口位置

**决策**: 在 User 组件（退出登录上方）增加"外观设置"菜单

**理由**:
- 与系统设置入口集中管理
- 符合用户操作习惯

### 4. 默认主题行为

**决策**: 新用户默认"跟随系统"

**理由**:
- 现代操作系统都支持暗色模式
- 提供开箱即用的体验

## Risks / Trade-offs

- **风险**: 硬编码颜色分散在多个组件中
- **缓解**: 先梳理所有样式文件，建立颜色变量映射表

- **风险**: 第三方组件可能不支持 Ant Design 主题
- **缓解**: 优先使用 Ant Design 原生组件
