## Context

当前应用需要用户先打开主窗口才能记录笔记，流程繁琐。用户希望在 macOS 托盘处直接触发快速笔记功能。

现有实现：
- 托盘功能已在 `electron/tray.ts` 中实现
- 笔记保存接口 `/note/add` 已在后端实现

## Goals / Non-Goals

**Goals:**
- 实现托盘"快速笔记"菜单
- 弹出快速笔记输入窗口
- 提交笔记到后端并关闭窗口

**Non-Goals:**
- 不修改主应用界面
- 不实现笔记分类、标签等高级功能（仅快速记录）

## Decisions

1. **快速笔记窗口实现方式**: 使用 BrowserWindow 弹出独立窗口
   - 理由：React 组件需要在独立窗口中渲染，BrowserWindow 更轻量且与 Electron 集成更好

2. **数据传输方式**: 通过 IPC 通信
   - 理由：快速笔记窗口与主进程分离，需要通过 IPC 传递数据

3. **笔记保存**: 复用现有 `/note/add` API
   - 理由：减少重复开发，保持接口一致性

## Risks / Trade-offs

- [风险] 快速笔记窗口样式与主应用不一致
  -  mitigation: 复用主应用的样式和组件
- [风险] 用户未登录时无法保存笔记
  - mitigation: 快速笔记可以使用默认用户或提示登录
