# Nexa - AI 知识库桌面应用

## 需求功能
- 系统中有上传图片，文档等功能，目前存在问题：
  - 上传图片和文档是不同的controller中实现，且代码高度一致，需要抽离，保留在common-controller中
  - 上传的图片和文档都在electron应用的本地，换个电脑后文件就找不到了，想同步一份到云端，以云端返回的链接为基准
- 在前端展示层面也是存在拼接返回地址的情况，需要优先使用云端地址，云端失败才是本地，而且我希望图片加载一次就缓存，不需要多次去请求服务端
- 上传文档/图片的逻辑：先本地上传，后同步云端，如果上传本地都失败了要求重试，同步云端失败优先用本地的链接，后续静默实现云端同步和更新数据
- 在已有的图片表/文档表需要有本地连接和云端连接两个字段，用于同步云端失败后，后续静默同步好了实现安装本地路径去更新这两个表中的云端链接字段
- 同步云端单独成一个service，同步云端也是一个普通的http接口，需要一个key字段来做校验，防止恶意调用
