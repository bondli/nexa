# Nexa - AI 知识库桌面应用

## 需求功能
- 新增报告页面
- 具体功能点：
  - 在桌面端，左侧菜单中，目前已有的最后一个icon，点击后进入报告页面（已实现）
  - 报告页面UI布局：
    - 左侧分类导航，参考图片`Picture`页面，分成两个部分，1：有固定的两个菜单：全部报告，全部月报；2：报告分组，下面是按月一个个菜单，如：2026-05、2026-04、026-03、2026-02...倒序展示有日报的每个月
    - 右边主要内容，参考文章`Article`页面，是根据左侧的点击对报告列表做的筛选，报告列表每条记录展示：固定Icon(GithubFilled)，报告日期，报告类型，报告摘要，报告创建时间
      - 点击报告日期和报告摘要等通过Drawer右侧拉出抽屉展示报告详情
      - 点击报告固定的Icon，可以展示报告的缩略图
  - 报告页面功能:
    - 报告列表，支持分页加载
    - 支持新增日报/月报，每天写的日报是昨天的，每个月初写的上个月的月报，点击新增按钮（下拉出现日报、月报两个选项），点击出现Modal框，提示在生成中，调用生成报告接口，自动生成报告，生成完成之后将报告内容使用markdown预览器呈现在Modal框内
    - 生成报告的接口是需要区分是写日报还是月报
    - 写日报：需要根据昨日产生的笔记和文章，总结出来：1、核心的内容和知识点，2、提示用户今天可以跟进的事项和学习的知识（调用LLM来生成）
    - 写月报：需要根据上月产生的笔记和文章，总结出来：1、核心的内容和知识点，2、提示用户本月可以跟进的事项和学习的知识（调用LLM来生成）
    - 周报月报的模版大概是五部分：1、核心记录的笔记内容；2、主要完成的工作；；3、所收藏的文章的核心知识点；4、总结过去一天/一月的知识收获；5、后续重点可以跟进的事项
  - 进入页面需要检测：
    - 当天是否已经有了昨天的日报，如果没有，提示用户需要写日报
    - 当前是否已经是月初（每个月的5号前定义是月初，10月是10号前），如果是月初，并且上个月的月报没有，提示用户写月报
  - 新建一个报告的数据表（Report），有一个字段是reportType（枚举值，日报/月报），该数据表有的字段：
    - reportDate: 报告日期 string
    - reportType: 报告类型 'daily' | 'monthly'
    - summary: 报告摘要 string
    - content: 报告内容（markdown格式） text
    - image: 报告缩略图 string
    - createdAt: Date
    - updatedAt: Date
  

### 目标
- 新增报告页面，支持新增日报和月报，支持查看报告列表和详情，日报月报的生成，以及对应日报月报的图片生成
