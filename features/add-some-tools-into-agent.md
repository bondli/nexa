# Nexa - AI 知识库桌面应用

## 需求功能
- 增加几个工具内置到agent中
- 具体功能点：
  - 目前在./server/services/agent/tools/builtins.ts中内置了几个工具，但是都是没有实现具体功能的，需要你来实现具体功能
    - 具体需要实现的工具：
    - write_note：创建笔记，需要调用实际的笔记服务创建笔记，当用户没有提及在哪个分类下创建的时候，你需要通过对话方式确认用户想要创建到哪个分类中，需要查询到所有分类给用户选择
    - search_notes：搜索笔记，需要调用实际的搜索服务，实现搜索，并在返回查询到的笔记列表，用户选择其中的一条的时候需要查询给到详细的笔记内容
    - get_weather：查询天气，需要调用实际的天气API接口来查询，可以找国内免费的api来实现
  - 新增一个增加闹钟的工具内置到agent中，具体如下：
    - alarm_clock：增加闹钟，需要调用实际的系统闹钟接口来设置闹钟，到点了需要有桌面通知提醒
  - 新增文章查询工具
    - search_articles：搜索文章，需要调用实际的文章搜索服务，实现搜索，并在返回查询到的文章列表，用户选择其中的一条的时候需要查询给到详细的文章内容

### 目标
- 完善已有的三个工具：write_note、search_notes、get_weather
- 新增工具：alarm_clock、search_articles
- 所有新增的工具都需要支持在chat页面直接调用，不需要用户手动输入工具名称，缺少参数的时候需要通过对话方式让用户补全参数，然后继续

### 上下文
- 内置工具位置： ./server/services/agent/tools/builtins.ts
- 后端：
  - notes相关接口：./server/controllers/note-controller.ts
  - articles相关接口：./server/controllers/article-controller.ts


### 验收标准
- 进入桌面端，进入聊天页面，发起新会话，可以直接调用工具，无需用户手动输入工具名，如：
```
帮我记一下：明天上午9：00有一个重要的会议，和小明沟通后续团队招聘事宜
```
  - 系统需要能够理解用户的意图，并自动调用相应的工具（write_note）来完成任务
  - 上述对话，缺少笔记保存的分类，应该提示用户选择分类，查询到笔记分类让用户选择
  - 用户可以通过回复分类的数字进行选择
  - 最后提示工具调用的结果
